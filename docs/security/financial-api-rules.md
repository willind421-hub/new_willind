# 금융 API 보안 규칙

자동매매, 주식, 은행 등 돈과 직결된 API 연동 시 반드시 따를 규칙.

---

## 1. API 키 관리

- **절대 금지**: 채팅 채널(Claude Code, 디스코드, 대시보드)에 API 키/시크릿 값 언급
- **저장 위치**: `.env` 파일에만 저장, 코드에 하드코딩 금지
- **Git**: `.env` 반드시 `.gitignore`에 포함 — 커밋 금지
- **주기적 교체**: 3개월마다 또는 의심 상황 즉시 재발급

## 2. 조회 / 거래 키 완전 분리

| 용도 | 권한 | 저장 위치 |
|---|---|---|
| 대시보드 모니터링 | 조회 전용 (read-only) | `dashboard/.env` |
| 자동매매 실행 | 거래 권한 | 별도 서버 `.env`, 절대 대시보드와 공유 금지 |

- 대시보드 서버가 털려도 거래 키는 분리되어 있어야 함
- 조회 키로는 주문 자체가 불가능한 구조

## 3. IP 화이트리스트 (핵심 방어선)

연동 시 가장 먼저 설정:
- 한국투자증권 OpenAPI: 앱 등록 시 IP 등록 기능 제공
- 키움 OpenAPI: 사용 PC IP 고정 등록
- 그 외 증권사/거래소 API: 동일하게 서버 IP만 등록

→ 키가 유출돼도 **등록된 IP에서만 호출 가능** — 해커 서버에서 쓸 수 없음

## 4. 일일 거래 한도

- API 설정 또는 코드 레벨에서 일일 최대 거래 금액/횟수 하드 제한
- 이상 거래 감지 시 자동 중단 로직 필수

## 5. 모니터링

- 모든 API 호출 로그 저장 (주문 체결/실패 포함)
- 비정상 호출 감지 시 즉시 모니터링부 알림
- 평소 패턴 벗어난 대량 주문 → 자동 정지 + 사용자 알림

## 6. 침해 발생 시 즉시 대응

1. 해당 API 키 즉시 revoke (플랫폼 앱 관리 페이지)
2. 증권사/은행 고객센터 즉시 신고
3. 비밀번호 전체 교체
4. 피해 범위 파악 후 금융감독원 신고 (필요 시)

---

## 운영자/역할 행동 규칙

- 금융 API 연동 작업 시작 전 이 파일 반드시 읽기
- 디버깅 중 `.env` 읽을 때 → Grep으로 키 이름만 확인, 값 출력 금지
- 금융 API 관련 오류 메시지에 키 값 포함 시 → 로그에 마스킹 처리 후 보고
- 사용자 컨펌 없이 거래 권한 키를 어떤 채널에도 노출 금지
